Nicotine+ on Slax




Nicotine+ Modules

Download latest stable modules @ sourceforge.net
Unstable modules you can download @ osiris.network

Required modules

  • Python 2.4 module for Slax
  • PyGTK 2.6.2 module for Slax
  • GTK+2 2.6.8 module for Slax

Optional modules

if you want others to see the bitrate of ogg/vorbis files you're sharing
  • python-ogg 1.3-2 module for Slax
  • python-vorbis 1.3-2 module for Slax (if you want others to see the bitrate of ogg/vorbis files you're sharing)
if you want to block users from specific countries
  • GeoIP C Libary 1.3.16-1 module for Slax
  • python-geoip 1.2.16-1 module for Slax

Installation

Download required modules and put them in /modules of Slax. If you want, you can also download the optional modules and put them in /modules as well. Now go to you Slax folder: 
root@slax:~# cd /path/to/slax/folder
Generate the modified Slax: 
root@slax:/path/to/slax/folder# ./make_iso.sh
Burn on CD and enjoy!

Another way

Another way how to use Nicotine+ SLAX module is by executing uselivemod command. Syntax is following: 
root@slax:~# uselivemod /path/module.mo
The module will be inserted into live filesystem immediately, so the
application will look like it would be installed. You will need to run
the application by apropriate command. Unfortunately using modules on
the fly by uselivemod can sometimes hang your machine, it's some Slax
bug which needs to be fixed. It's NOT a Nicotine+ bug!!

If you are using SLAX Standard Edition (or any other edition with KDE) you can doubleclick the module.mo file in konqueror, this will do the trick too.
